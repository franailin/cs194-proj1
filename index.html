My initial approach was to use to divide each picture into the respective red blue and green, crop them, then use a double for loop to find the best alignment with SSD as a metric. For jpgs, I didn't build a pyramid and instead used a double for loop to loop through width/10 for x and height/10 for y, before translating them using np.roll with the best alignment. For tiffs, I built a pyramid by scaling each level by a factor of 1/2 and then updating the best alignment as I went down the pyramid. I tried to parallelize finding the alignments for red and green using the multithreaded library, but starmap would get stuck on the first loop, so I just used the default map function for python instead. It has the same performance as the double for loop, but it suggests the idea of parallelization.
